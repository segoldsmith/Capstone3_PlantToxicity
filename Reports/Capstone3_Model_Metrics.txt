==============================================================================
 CAPSTONE 3 - MODEL METRICS SUMMARY
 Poisonous vs. Non-Poisonous Plant Classification
==============================================================================

FINAL MODEL: Tuned EfficientNet-B0 (3-phase fine-tuned, calibrated, threshold-tuned)


------------------------------------------------------------------------------
1. MODEL ARCHITECTURE
------------------------------------------------------------------------------
Base architecture .............. EfficientNet-B0
Pretrained weights .............. ImageNet1K_V1
Classifier head .................. Linear layer, in_features -> 2 classes
Output classes ................... 2 (Non-Toxic = 0, Toxic = 1)
Input image size ................. 224 x 224 (resized from 256, center/random crop)
Framework ......................... PyTorch (torchvision.models)


------------------------------------------------------------------------------
2. INPUT FEATURES / PREPROCESSING
------------------------------------------------------------------------------
Input type ........................ RGB plant photographs
Normalization mean ................ [0.485, 0.456, 0.406]  (ImageNet stats)
Normalization std ................. [0.229, 0.224, 0.225]  (ImageNet stats)

Training-set augmentation:
  - Resize ........................ 256 px (shorter side)
  - RandomResizedCrop ............. 224 px, scale (0.8, 1.0)
  - RandomHorizontalFlip
  - RandomVerticalFlip
  - RandomRotation ................ +/- 20 degrees
  - ColorJitter .................... brightness=0.2, contrast=0.2, saturation=0.2
  - RandomAffine (shear=2) ........ applied with p=0.15
  - RandomPerspective (distortion=0.05) ... applied with p=0.10
  - RandomErasing ................. p=0.2, scale (0.02, 0.15), ratio (0.3, 3.3)
  - Mixup (alpha=0.4) ............. applied to ~25% of batches
  - CutMix (alpha=0.4) ............. applied to ~25% of batches

Validation-set preprocessing:
  - Resize (256) -> CenterCrop (224) -> Normalize (no augmentation)

Note: Heavy geometric augmentation (flips, rotation, cropping) was used
specifically to prevent the model from learning image ORIENTATION as a
shortcut for toxicity, after EDA found a statistically significant
association between orientation and toxicity label (chi-square = 58.62,
p < 0.001) in the raw dataset.


------------------------------------------------------------------------------
3. DATA SPLIT
------------------------------------------------------------------------------
Total images ...................... 9,952
Train / Validation split .......... 80% / 20%
Split method ....................... random_split (seed = 42)
Batch size ......................... 32
Random seed ........................ 42 (set across random, numpy, torch)


------------------------------------------------------------------------------
4. TRAINING HYPERPARAMETERS (3-Phase Fine-Tuning)
------------------------------------------------------------------------------
Loss function ...................... CrossEntropyLoss
  - Class weights ................. [1.0 (non-toxic), 1.3 (toxic)]
  - Label smoothing ................ 0.05
Optimizer .......................... Adam

  Phase 1 - Train classifier head only (backbone frozen)
    Epochs ........................ 6
    Learning rate ................. 2e-4
    LR schedule ................... Linear warmup (3 epochs) -> Cosine annealing

  Phase 2 - Unfreeze top 30% of backbone (blocks 5-6)
    Epochs ........................ 4
    Learning rate ................. 3e-5
    LR schedule ................... Linear warmup (2 epochs) -> Cosine annealing

  Phase 3 - Full fine-tune (entire network unfrozen)
    Epochs ........................ 10
    Learning rate ................. 1e-5
    LR schedule ................... Cosine annealing (T_max=10, eta_min=5e-6)

Total training epochs .............. 20 (across all 3 phases)
Total training time ................ ~178 minutes (~3.0 hours)


------------------------------------------------------------------------------
5. CALIBRATION & DECISION THRESHOLD
------------------------------------------------------------------------------
Calibration method ................. Temperature scaling (post-hoc, on validation logits)
Learned temperature ................ 0.828
Threshold tuning method ............ Swept thresholds 0.10-0.90; selected value
                                      maximizing F2 score (recall-weighted)
Operating decision threshold ....... 0.28
                                      (toxic predicted when P(toxic) >= 0.28)

Rationale: Default threshold (0.50) reached only 86.4% recall on the toxic
class. Threshold was lowered specifically to satisfy the project's primary
goal of >=90% recall, accepting a precision trade-off (more false-positive
toxic flags) in exchange for far fewer missed toxic plants.
